Virtual Machines and Containers

VM isolates the hardware

Container isolates the process

On the infrastructure side, if you look at deployment and density options provided by containers compared to VMs, instead of recreating unique hardware, kernel address, resources, etc for each machine, containers share a host kernel - can be bare metal or a VM, but otherwise the isolation is provided closer to the application layer.

Containers aren't really a new thing btw - modern containers just provide clean and user friendly interfaces to system-calls and kernel functions that date back decades.

Kubernetes

So what is this Kubernetes thing? It's an open-source project originated at Google that is used to orchestrate and operate containers at scale. Hmm, that sounds like just the cure for what ails us!
The idea behind K8s is that it will take your container workloads that have been declaritively described and run them amongst a cluster of compute. It can help do some basic connecting of containers with an internal DNS service, and it's all based on a REST API which makes interacting with this container platform very easy.
Naturally fits into a DevOps lifecycle, developer submits code, CI/CD engine pushes out a container, Kubernetes runs your containers on your hardware! Sweet!
Well, before we call it a day, we still need some internal networking for nodes and container pods to address each other. Ok, no problem.
Well, we need somewhere to store the images we built, or proxy them so we're not pulling from external resources over the WAN every time for every container...ok, that shouldn't be hard to integrate, right?
Oops, don't forget we need to do the DevOps-y thing and get some feedback and metrics into what's going on. How are we to know the container hasn't entered a CrashLoop state?
What happens when it does enter a CrashLoop state? How do you manage that lifecycle, and the lifecycle of scalaing and rolling deployments?
Now at this point we're building integrations and so we need our own application services to glue all this together, yay we're building systems to run systems...
